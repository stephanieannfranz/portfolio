design

With design, my main concerns are accessibility and user-friendliness. I think about my parents when I’m designing - will the design be easy for them to use and read? If not, that design should be trashed. Of course I want the design to be beautiful and pleasing, but a beautiful design is useless if users either can’t use or struggle to use the product. I’m also interested in designing components that can be used across devices and apps, rather than designing a complete “website”.

  • Branding and Identity Design
  • Color, Typography, and Accessibility
  • Sketching, Wireframing, and Prototyping
  • Style Guides, Style Tiles, Atomic Design, and Pattern Libraries
  • UI and UX Design, User Personas, and User Testing
  • Adobe Photoshop, Illustrator, and Experience Design
  • Mobile-first and Responsive Web Design
  • Skillcrush Visual Designer Blueprint
  • Skillcrush Web Designer Blueprint

development

I like code that is clean, efficient, fast, and, most importantly, does what I want it to do. My approach is to plan out each component, then build the components one at a time, solving any problems that arise along the way. I follow the principles of progressive enhancement, allowing me to use new technology and frameworks while also ensuring that the website is usable across browsers and devices.

  • HTML
  • CSS (Box model, Flexbox, Grid, BEM Methodology, Sass, animation)
  • JavaScript (vanilla JS, currently learning jQuery and Node JS)
  • WordPress (theme creation/customization, child themes, plugins, PHP)
  • Grunt (responsive images, image optimization, CSS post-processing, watch task)
  • Command line, Git, Github
  • Skillcrush WordPress Developer Blueprint
  • Udemy Web Developer Bootcamp
